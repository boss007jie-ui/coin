# CEX 实战交易宪章

这个项目不是行情玩具，不是科研看板，也不是为了堆指标。

它的核心目标是：把新山寨币、异动合约、无现货锚定币、创新高币种里的可交易机会，转化成可执行、可风控、可复盘、最终可自动化的交易动作。

## 项目性格

我们做的是实战交易系统。

所有监测都必须服务于未来赚钱，但赚钱不是靠冲动追涨，而是靠一套可重复执行的流程：

- 发现机会：从 Binance、Aster 或其他交易所里找到可能出现趋势、挤压、诱多、诱空、爆拉、暴跌的标的。
- 判断方向：把信号翻译成做多、做空、观望、回避，而不是只给一堆中性描述。
- 控制风险：合约默认逐仓，杠杆受限，必须有移动止损。
- 管理仓位：本金有限，仓位分配必须服务于账户生存，而不是单笔押注。
- 执行反馈：每一笔模拟或实盘交易都要知道为什么开、为什么平、盈亏多少、是否按计划执行。
- 复盘优化：用交易结果修正策略，而不是用更多指标掩盖错误。

## 最高优先级

1. 保住本金
2. 捕捉高赔率机会
3. 避免高风险噪音
4. 缩短从信号到动作的路径
5. 用真实结果淘汰无效规则

如果一个功能不能帮助以上至少一项，它就不是优先事项。

## 交易默认原则

- 合约使用逐仓，不使用全仓。
- 每一笔合约必须有移动止损。
- 杠杆默认不超过 `5x`。
- 小资金账户优先控制回撤，而不是追求一次翻倍。
- 信号反转时允许主动平仓，不傻等止盈或止损。
- 风险恶化时先离场，再讨论是否重新入场。
- 任何自动化交易前，必须先在模拟盘里验证。

## 当前模拟盘原则

- baseline 组：保守止盈，`+8% ~ +18%` 取 `+8%`。
- optimistic 组：乐观止盈，`+8% ~ +18%` 取 `+18%`。
- 两组都使用逐仓和 `6%` 移动止损。
- 两组独立记账，用结果比较策略，而不是靠感觉争论。
- 本金低于 `500 USDT` 时触发复盘并切换防守策略。

## 我们不做什么

- 不为了好看而做页面。
- 不为了显得专业而堆指标。
- 不做无法转化成交易动作的监测。
- 不把“观察”当成最终结果。
- 不让全仓、无止损、高杠杆进入默认策略。
- 不因为一次赚钱就提高风险上限。
- 不因为一次亏损就否定整个系统，要看样本、规则和执行质量。

## 每个新功能必须回答的问题

1. 它能帮助我们更早发现机会吗？
2. 它能帮助我们更准确判断多空吗？
3. 它能帮助我们更好控制风险吗？
4. 它能帮助我们更快执行或提醒吗？
5. 它能帮助我们复盘并优化策略吗？
6. 它会不会让系统变得更复杂但不更赚钱？

如果答案主要是第 6 条，这个功能应该暂缓。

## 项目北极星

最终目标不是“看懂市场”，而是建立一套能在真实市场里长期活下去、持续发现机会、严格控制亏损、不断优化胜率和赔率的合约交易系统。
